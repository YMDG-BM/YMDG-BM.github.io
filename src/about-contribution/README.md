知识的火花因传递而燎原，技术的围墙因分享而崩塌。这里不是一个单向输出的讲台，而是一块等待共同开垦的沃土——无论你是初涉安全的新人，还是身经百战的专家，只要心怀热爱，你就是这片知识花园的共建者。

## 🌱 你可以贡献什么？
- **CTF 题解与技巧**：分享你的解题思路、新奇姿势或参赛感悟，让虚拟战场的旗帜指引更多人。
- **Wiki 知识条目**：将零散的学习笔记升维为稳定、可复用的确定性知识，填补概念网络的空白或修正已有偏差。
- **运维实战经验**：记录部署、监控、排障、自动化的血泪教训与优雅方案，让系统工程少走弯路。
- **任何形式的优化**：修正错别字、优化排版、补充引用、更新失效链接……细节的打磨同样珍贵。

## 🛠 如何贡献？
1. 通过 **GitHub**
本博客内容托管于 [GitHub 仓库](https://github.com/YMDG-BM/YMDG-BM.github.io)。你可以：
- 提交 Issue：指出错误、提出建议或申请新主题。
- 发起 Pull Request：直接修改或新增文章，我们会及时审阅合并。
2. 通过内部联系
- 如果你是 SLsec 实验室 成员，可直接在团队群内 @ 或者私信管理员提交草稿。

## 📝 风格与规范
为了让知识呈现更加统一、舒适，请参考**已有文章的风格**：

尽量使用**中文**书写，术语首次出现时可附上英文原名（如“跨站脚本（XSS）”）。

**代码块标明语言**，关键步骤配以必要解释。

保持客观严谨，适当融入个人经验与思考。不过没有必要过于死板。

如有引用外部资料，请附上**可访问的链接**。

不必担心“写得不够好”——所有提交都会经过审阅和协作优化。迈出第一步，就是最好的开始。

## 📛 严格禁止的内容
为确保本博客的安全、合法与良好氛围，**严禁提交或包含以下任何内容**：

1. **违法违规信息**
包括但不限于：**侵犯他人隐私、传播恶意软件/后门、教唆犯罪、发布未授权的漏洞利用代码（无防御目的）、色情、暴力、赌博、仇恨言论或任何违反所在地法律的内容**。

2. **抄袭与侵权**
**直接复制他人文章、代码或题解而不注明出处，或未经授权使用受版权保护的素材。**所有贡献**必须尊重原创，引用须符合合理使用原则**。

3. **破坏性行为**
提交含有**恶意链接、XSS 载荷、反向 shell 等攻击性 payload 的 PR 或 Issue**；**恶意刷屏、破坏网站结构或试图利用博客平台漏洞**。

4. **人身攻击与不友善言论**
对**任何贡献者、读者或第三方进行辱骂、歧视、威胁或骚扰**。本博客坚持技术中立与平等对话。

5. **无关广告与灌水**
纯粹**推广商业产品、加密货币、传销、刷量等无技术价值的内容**；**重复提交无意义的占位符或乱码**。

违反上述规定的内容将被直接删除，并视情节严重程度采取：**警告、屏蔽贡献者、向 GitHub 或执法机构举报等措施**。
