如果说“纸上得来”关注的是解题、出题、开发与应急中的过程与手感，那么这里的“文章集”，便是那些过程最终凝结成的、可供他人阅读的完整叙事。它既是对“纸上得来”所述土壤的收成，也是让零散的草稿与折痕得以被整理、被呈现的展台。

与上层综述不同，这片区域不再谈论“如何思考”与“怎样犯错”的共性体验，而是存放每一次具体思考的终稿：一篇写完了的 Writeup、一份整理后的漏洞分析、一段沉淀下来的开发心得、或者一次应急事件结束后的复盘记录。它们比笔记更完整，比随想更严谨，却又比正式论文更贴近实战的温度。

之所以将这些文章单独聚集于此，是为了让左侧的导航保持清爽——技术脉络、项目文档、碎片随想各安其位，互不打扰。而当你真正想要细读某一篇完整的文字时，这里便是它们的归宿。你可以按标题翻阅，也可以随手点开一篇，不必担心被其他目录打断。

这些文章彼此独立，却在时间线上共同勾勒出一个技术学习者的成长轨迹：从迷茫到清晰，从零散到系统，从记录给自己到写给别人看。它们是“纸上得来”的具象化，也是这片数字花园里，最接近“终稿”形态的果实。
